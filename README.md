# 学术文献检索智能体研究 —— LLM增强型检索Query生成工具 README

# 项目介绍

本项目是**学术文献检索智能体研究**的核心支撑工具，基于通义千问API（qwen\-turbo）和原始 query：title + abstract开发，专注于提升学术文献检索的精准度与效率。核心功能是读取学术论文相关数据（支持全文/关键片段输入），通过大语言模型（LLM）生成结构化检索画像，进而输出增强后的检索Query，同时保存原始Query与LLM增强Query的对比结果，为学术文献检索智能体提供高质量的检索输入，助力智能体快速、准确定位目标文献。

项目核心优化：针对学术文献检索的专业性需求，将原有的“论文片段（标题、摘要、引言前2段、结论前1段）输入LLM”改为“论文全文输入LLM”；若未提供全文，则自动拼接完整摘要、引言、结论，确保LLM获取更全面的学术信息，生成更贴合学术检索场景、更具针对性的增强型Query，适配学术文献检索智能体的核心需求。

# 配置说明

核心代码中需配置以下关键参数（已在代码中标注，直接修改即可，适配学术研究场景）：

- **DASHSCOPE\_API\_KEY**：通义千问API密钥，需前往 [通义千问控制台](https://dashscope.aliyun.com/) 申请，替换代码中默认的密钥；建议妥善保管，避免泄露（学术研究中需注意数据安全与隐私保护）。

- **PROMPT\_PATH**：LLM提示词模板（llm\_prompt\_v1\.md）的路径，若修改了模板文件名或存放路径，需同步更新该参数，确保模板能正常加载。

- **COMPARE\_JSON**：Query对比结果文件的保存路径，可根据学术研究的实验需求，修改保存目录和文件名，便于后续实验数据的整理与分析。

注意：API密钥属于敏感信息，请勿泄露或提交至公开代码仓库，建议通过环境变量方式管理（可选优化，符合学术研究中数据安全规范）。
